### 简介

虽然大模型简化了很多数据采集操作、提升了互联网数据消费热情。缺点是不能自动获取复杂网站的数据；有的能获取但很不稳定；有些能直接使用大模型自动化获取数据，但消耗token较多；更不用说进行复杂的代理管理、多账号与登录管理、流量控制、去重、重试、监控、加密字体解密等。

以往大家主要针对特定需求场景，使用scrapy开发合理使用代理、队列实现大量、并发数据采集；如果需求复杂往往需要单独开发，如复杂的流量控制、多账号与登录管理、字体加密等。

letsscrapedata支持以上[数据采集工具的所有特性](https://letsscrapedata.github.io/scraper/#/scraping/feature)，可大大简化或减少数据采集开发、维护、部署、使用成本。

主流自媒体平台作为数据采集常见的学习、关注平台，有很多python、JavaScript等开发语言实现的学习仓库。本仓库主要借鉴python开源仓库[MediaCrawler](https://github.com/NanmiCoder/MediaCrawler)，实现letsscrapedata模板版本，所有模板可使用[开源包](https://github.com/LetsScrapeData/scraper)或APP(https://www.letsscrapedata.com/)执行。
